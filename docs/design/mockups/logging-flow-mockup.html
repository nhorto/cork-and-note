How the hybrid works

There's one underlying object — a logging session (today's "visit", generalized). It holds 1…N wines and an optional place.

  • BLog one wine, save. The session has a single wine and (usually) no place. Feels like a quick wine entry.
  • AAdd another wine. The same session now groups several wines under one place/date — the winery-trip recap you have today.
  • ↺Start at a winery. Tapping a winery (map/Home) opens a session pre-set to that place; you just keep adding wines.
  • ✎Editable later. Add a place, add wines, or fix details after saving — including turning a one-wine log into a session.

Open question for you: should the entry point default to “Log a wine” (B-first, add place/wines as you go) or also offer an explicit “Start a winery visit” shortcut? The mockup shows both coexisting.
